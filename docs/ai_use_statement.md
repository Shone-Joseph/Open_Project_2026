# AI Use Statement

## Scope of AI assistance

Generative AI was used in a limited and supervised manner as a technical and editorial assistant during this coursework project.

Its role was restricted to supporting tasks such as:

* troubleshooting Python and Jupyter errors;
* suggesting clearer folder and file organisation;
* improving code readability;
* helping standardise file paths;
* proposing documentation structure;
* assisting with wording and formatting;
* identifying possible reproducibility issues;
* suggesting checks for expected outputs.

AI was not used to replace the research process, independently conduct the analysis, or make final interpretive decisions.

## Research decisions and authorship

The project author remained responsible for:

* defining the research question;
* selecting the Europeana search strategy;
* determining the query groups and search terms;
* deciding how the data should be cleaned and deduplicated;
* selecting the analytical variables;
* reviewing Wikidata matches;
* interpreting the results;
* identifying limitations;
* preparing the final coursework submission.

All substantive research decisions were made by the author.

## Code use and verification

Where AI suggested or revised code, the code was manually reviewed, inserted, executed, and tested by the author.

The author verified:

* whether the code ran successfully;
* whether the correct files were loaded;
* whether outputs were saved to the intended locations;
* whether dataset dimensions matched the expected values;
* whether figures and tables reflected the underlying data;
* whether errors or inconsistencies were present;
* whether the final workflow could be reproduced from the repository.

AI-generated code was treated as a draft requiring validation, not as an authoritative result.

## Data and analysis

The actual dataset was collected through the Europeana API and stored in the project repository.

All analysis was performed on the project data using the documented Jupyter notebooks.

The author manually checked key results, including:

* 1,500 raw records;
* 245 deduplicated records;
* the creation of transformed variables;
* the generation of figures and summary tables;
* the final Wikidata-enriched dataset;
* the final match-status counts.

## Wikidata enrichment

AI assistance was used only to help organise the enrichment workflow and improve the structure of the code and documentation.

Provider matches were reviewed manually.

Where a Wikidata match could not be confirmed confidently, it was labelled as:

* `uncertain`, or
* `not_checked`

rather than being presented as a confirmed result.

The final expected match-status counts were taken from the executed notebook output:

* matched: 139
* uncertain: 28
* not_checked: 78

## Documentation assistance

AI was used to assist with wording, structure, and clarity in project documentation.

All documentation was reviewed and edited by the author to ensure that it accurately reflected:

* the actual repository structure;
* the correct filenames;
* the executed notebook workflow;
* the real outputs;
* the project limitations;
* the author’s own interpretation.

## Monitoring and quality control

AI assistance was continuously monitored.

Suggestions were accepted only when they were consistent with:

* the actual project files;
* the notebook outputs;
* the dataset;
* the course requirements;
* the author’s own understanding of the research process.

Incorrect, unclear, or unsuitable suggestions were revised or rejected.

## Academic responsibility

AI functioned only as an assistant for technical support, troubleshooting, and editorial improvement.

The author retains full academic responsibility for:

* the originality of the project;
* the correctness of the code;
* the validity of the analysis;
* the interpretation of the results;
* the ethical framing;
* the final submitted work.

This statement is included to ensure transparency about the limited and supervised use of AI during the project.
